# АІС "Магазин автозапчастин"

Багатосервісна інформаційно-аналітична система для управління магазином роздрібної торгівлі автозапчастинами. Проєкт розроблено в рамках лабораторного практикуму з **DevOps (IaaS розгортання)**.

Система дозволяє вести статистику продажів, контролювати складські запаси, аналізувати ефективність постачальників та формувати складні аналітичні звіти.

Виконала Куліченко Софія з групи ІО-32

## Архітектура проєкту (Рівні)

Проєкт побудований за класичною багатошаровою (**N-Tier**) мікросервісною архітектурою з чітким розділенням на **Stateful** та **Stateless** компоненти:

1. **Frontend (UI / Відображення)** — *Stateless*
   - Односторінковий додаток (SPA) у вигляді дашборду. Відповідає виключно за рендеринг даних та взаємодію з користувачем.
2. **Backend (API / Бізнес-логіка)** — *Stateless*
   - RESTful API, що обробляє HTTP-запити від клієнта, виконує складні аналітичні SQL-запити до бази даних та керує кешуванням. Готовий до горизонтального масштабування.
3. **Database (Зберігання даних)** — *Stateful*
   - Реляційна база даних, що зберігає інформацію про номенклатуру, постачальників (фірми, дилери, дрібні виробництва), замовлення та продажі.
4. **Cache (Кешування)** — *Stateful / Ephemeral*
   - In-memory сховище ключ-значення. Використовується для кешування результатів "важких" аналітичних запитів з метою зниження навантаження на основну БД.

## Інструкція із запуску (Локальне середовище)

### Передумови (Prerequisites)
Переконайтеся, що на вашій машині встановлені:
* [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
* [Node.js](https://nodejs.org/) 
* СУБД PostgreSQL (локально або віддалений доступ)
* Redis Server (на Linux/WSL або Memurai для Windows)

### 1. Налаштування Бази Даних
1. Виконайте SQL-скрипти з папки `/database` у вашому екземплярі PostgreSQL для створення таблиць та наповнення їх тестовими даними.
2. Переконайтеся, що користувач має права на виконання операцій `SELECT`, `INSERT`, `UPDATE`.

### 2. Запуск Backend (.NET API)
1. Відкрийте термінал у папці бекенду (там, де знаходиться файл `.csproj`).
2. Відкрийте файл `appsettings.json` і налаштуйте рядки підключення:
   ```json
   "ConnectionStrings": {
     "OracleDbConnection": "User Id=ВАШ_ЮЗЕР;Password=ВАШ_ПАРОЛЬ;Data Source=localhost:1521/XEPDB1;",
     "RedisConnection": "localhost:6379"
   }
3. Відновіть залежності та запустіть проєкт:
   ```bash
   dotnet restore
   dotnet run
API буде доступне за адресою: https://localhost:7001 (або http://localhost:5000). 
Також доступний Swagger UI за шляхом /swagger.

### 3. Запуск Frontend (React + Vite)
1. Перейдіть у папку фронтенду (vite)
2. Встановіть NPM-пакети:
   ```bash
   npm install
3. Переконайтеся, що у файлі конфігурації API (наприклад, src/api/client.ts) вказано правильний порт бекенду.
4. Запустіть сервер розробки:
    ```bash
   npm run dev
Дашборд відкриється у браузері за адресою http://localhost:5173.

### Доступний функціонал (Аналітичні запити)

- Фільтрація постачальників за категорією, обсягом поставок та періодом.
- Деталізація постачання конкретних деталей.
- Аналіз покупців за видами товарів та витраченими коштами.
- Моніторинг складських запасів (обсяг та локація).
- Кешовані звіти (Redis):
  - Топ-10 найбільш продаваних деталей та найдешевших постачальників.
  - Графіки середньомісячних продажів.
- Фінансовий аналіз: Розрахунок частки постачальника від загального обороту та прибутку магазину.
